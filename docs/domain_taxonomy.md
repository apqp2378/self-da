# docs/domain_taxonomy.md — 도메인·세부직무 분류 기준

> 근거: 업로드 자료 `데이터분석가_실무백서`(스파르타 내일배움캠프), `데이터 분석 공채 핵심 전략`, `직무 이해 답변`.
> 본 자료는 운영 보조 지표다. 토픽 라벨·직군 매핑은 사람이 최종 검수한다 (원칙 7).

## 1. 데이터 분석 도메인 (실무백서 기준)

| 도메인 | 대표 분석 대상 |
|---|---|
| 이커머스 | 클릭률, 장바구니 이탈, 구매 전환율, 재구매율 |
| 핀테크/금융 | 신용 분석, 리스크 모델링, 결제 패턴, 이상거래 탐지 |
| 헬스케어/의료 | 진료 기록 분석, 질병 예측, 병원 운영 최적화 |
| 마케팅 | 캠페인 분석, 유입/전환 퍼널, LTV |
| 게임 | 유저 행동, 튜토리얼 이탈률, 과금 패턴 |
| HR/People | 이직률, 조직 생산성, 인재 유지 예측 |
| 교육(EdTech) | 학습 이탈률, 성취도, 콘텐츠 추천 |
| 물류/공급망 | 배송 시간 예측, 재고 회전율, 경로 최적화 |
| 제조 | 품질 이상 탐지, 공정 개선, 예지보수 |
| 공공/정책 | 범죄율, 교통 흐름, 정책 효과 분석 |

## 2. 세부 직무 (실무백서 6분류)

Product · Marketing · Business · Operations · Financial · HR/People 데이터 분석가.

## 3. 프로젝트 enum 과의 매핑

Bullet Bank `직군 적합도` 및 Module A 토픽 라벨 후보를 위 도메인/세부직무에서 역산한다.

| 프로젝트 enum (직군 적합도) | 대응 도메인·세부직무 | Module A 토픽 라벨 후보 |
|---|---|---|
| 이커머스 | 이커머스 도메인 + Product/Business | 이커머스 그로스 분석 |
| 마케팅분석 | 마케팅 도메인 + Marketing 분석가 | 마케팅·캠페인 분석 |
| 플랫폼BI | Business/Operations + BI 도구 중심 | 플랫폼 BI·지표 운영 |
| 스타트업제너럴리스트 | 도메인 횡단 + 소규모 조직 | 스타트업 제너럴 분석 |
| 인하우스리서치 | Financial/HR/공공 + 리서치 성격 | 인하우스 리서치 |

> 주의: 이 매핑은 BERTopic 토픽을 사람이 라벨링할 때의 **후보 사전**이다. 실제 토픽은 데이터에서 나오며,
> 매핑되지 않는 토픽은 unknown으로 남긴다 (원칙 7).

## 4. jd_raw 수집 시 참고

- `industry` 값은 위 1번 도메인 표의 명칭으로 통일하면 후속 집계가 쉽다.
- `job_category` 값은 위 2번 세부직무 또는 공고 원문 직군명을 그대로 기록한다.

## 5. 신입 DA 필수 역량 (공채 전략 기준, 참고용)

기술: SQL · Python · 통계 기초 · 대시보드(Tableau). 자격증: SQLD · ADsP 권장.
비기술(GPT가 대체 못 하는 영역): 문제 정의 · 관점 설정 · 스토리텔링 · "왜?"를 끝까지 파는 사고력.
